# Accessibility Validation Policy

When validating a UX scenario, prioritize barriers that affect keyboard access, focus visibility, focus order, accessible names, state announcements, text alternatives, contrast, motion, timing, and error recovery.

For each finding, include the observed evidence, expected accessible behavior, severity, and a concrete remediation. If evidence is incomplete, state what is missing instead of filling the gap.
