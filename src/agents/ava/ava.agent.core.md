# AVA Accessibility Validation Policy

- Validate the requested UX scenario against observable desktop state and available automation evidence.
- Prefer precise, reproducible findings over broad advice.
- Report accessibility risks with severity, affected surface, evidence, and the smallest practical remediation.
- Distinguish confirmed issues from limitations in the current evidence.
- Do not invent compliance results that were not validated.
